*版本：v1.0*
*日期：2026年7月*

## 一、页面清单

LiveFront v2.0 共有以下页面/状态：

| 页面 | 说明 | 状态 |
|---|---|---|
| P1: 欢迎页 | 未打开项目时显示 | 默认状态 |
| P2: 主工作区 | 打开项目后显示 | 核心页面 |
| P3: 设置弹窗 | 用户配置设置 | 弹窗覆盖 |

## 二、P1: 欢迎页

### 2.1 页面状态

```
┌─────────────────────────────────────────────┐
│                                             │
│              [LiveFront Logo]               │
│                                             │
│           AI 是笔，LiveFront 是画板。        │
│                                             │
│  ┌──────────────┐    ┌──────────────┐       │
│  │   新建项目    │    │   打开项目    │       │
│  │  AI 从零生成  │    │ 打开已有文件  │       │
│  └──────────────┘    └──────────────┘       │
│                                             │
│          最近打开的项目列表（如有）           │
│                                             │
└─────────────────────────────────────────────┘
```

### 2.2 交互流程

```
用户打开 LiveFront
  → 显示欢迎页
  → 用户看到两个按钮：「新建项目」「打开项目」
  
场景 A：用户点击「新建项目」
  → 跳转到主工作区（P2）
  → 预览区显示空白页
  → AI 对话区自动激活
  → AI 询问："你想做什么网站？"
  → 用户输入描述
  → AI 生成代码 → 预览区显示页面

场景 B：用户点击「打开项目」
  → 弹出系统文件夹选择对话框
  → 用户选择文件夹
  → 跳转到主工作区（P2）
  → 文件树显示项目文件
  → 预览区渲染 index.html（或第一个 HTML 文件）

场景 C：用户点击最近打开的项目
  → 直接跳转到主工作区（P2）
  → 加载该项目
```

### 2.3 交互细节

| 元素 | 操作 | 行为 |
|---|---|---|
| 「新建项目」按钮 | 点击 | 跳转主工作区 + AI 对话激活 |
| 「打开项目」按钮 | 点击 | 弹出文件夹选择对话框 |
| 最近项目列表项 | 点击 | 加载该项目 |
| 最近项目列表项 | 右键 | 弹出菜单：移除最近记录、打开文件夹位置 |
| Logo | 点击 | 无操作 |

### 2.4 空状态设计

- 没有最近项目时，隐藏最近项目列表
- 窗口最小尺寸：800x600

│        │                                      │          │
│        │──────────────────────────────────────│          │
│        │  底部工作流拓扑图                       │          │
├────────┴──────────────────────────────────────┴──────────┤
│                    状态栏                                  │
└──────────────────────────────────────────────────────────┘
```

### 3.2 主工作区交互流程（完整）

#### 流程 A：新建项目 → 从零搭建

```
1. 用户从欢迎页点击「新建项目」
2. 进入主工作区
3. 文件树显示空项目结构
4. 预览区显示空白页
5. 右侧面板显示「AI」Tab
6. AI 对话区显示欢迎消息："你好！告诉我你想要什么样的网站。"
7. 用户输入："帮我做一个个人作品集，暗色主题"
8. AI 开始流式输出 → 对话区显示文字
9. AI 输出包含代码块 → 自动应用到项目文件
10. 预览区自动刷新 → 显示生成的网页
11. 底部拓扑图自动分析 → 显示模块结构
12. 用户看到效果
```

#### 流程 B：编辑已有网站

```
1. 用户从欢迎页点击「打开项目」
2. 弹出文件夹选择对话框
3. 选择项目文件夹
4. 文件树显示项目文件
5. 预览区渲染 index.html
6. 底部拓扑图自动分析 → 显示模块结构
7. 用户浏览页面
```

#### 流程 C：选中元素 → 修改样式

```
1. 用户在预览区右键点击一个元素（如导航栏）
2. 元素被选中 → 蓝色高亮框显示
3. 右侧面板自动切换到「属性」Tab
4. 属性面板显示该元素的 CSS 属性
5. 用户拖动「字号」滑块 → 从 14px 改为 18px
6. 预览区实时更新 → 导航栏文字变大
7. 用户继续修改其他属性
8. 或点击空白处取消选中
```

#### 流程 D：拓扑图交互

```
1. 用户在底部拓扑图中看到 [Header] → [Hero] → [Cards] → [Footer]
2. 用户点击 [Hero] 模块
3. 预览区自动滚动到 Hero 区域
4. Hero 区域高亮显示
5. 或用户发现 Hero 和 Cards 之间的连接有误
6. 用户在拓扑图中右键 [Hero] → 选择「编辑连接」
7. 拖拽连接线到正确的目标
8. AI 自动修改代码修复连接
```

#### 流程 E：AI 对话 → 修改代码

```
1. 用户在 AI 对话区输入："把导航栏背景色改成深蓝色"
2. AI 分析当前文件 → 找到导航栏的 CSS
3. AI 生成修改后的代码块
4. 对话区显示修改内容，带「应用」按钮
5. 用户点击「应用」
6. 文件被修改
7. 预览区自动刷新 → 导航栏变成深蓝色
8. 底部拓扑图更新（如结构有变化）
```

## 四、各模块交互详细设计

### 4.1 文件树（左侧）

**状态：**

| 状态 | 显示 |
|---|---|
| 未打开项目 | 空，仅显示标题"文件" |
| 打开项目 | 显示根文件列表 |
| 正在加载 | 显示加载指示器 |

**交互：**

| 操作 | 行为 |
|---|---|
| 单击文件 | 在预览区打开/刷新该文件 |
| 双击文件 | 在代码编辑器打开（高级模式） |
| 右键文件 | 弹出上下文菜单 |
| 右键空白处 | 弹出菜单：新建文件、新建文件夹 |

**上下文菜单：**

| 菜单项 | 操作 |
|---|---|
| 新建文件 | 弹出输入框，输入文件名后创建 |
| 新建文件夹 | 弹出输入框，输入文件夹名后创建 |
| 重命名 | 输入框编辑名称 |
| 删除 | 确认弹窗后删除 |

### 4.2 预览区（主体）

**顶部栏元素：**

**预览区交互：**

### 4.5 工作流拓扑图（底部）

**设计理念：**

拓扑图采用 ComfyUI 风格的节点式画布。每个模块是一个可拖动的大卡片，模块之间用贝塞尔曲线引导线连接。画布支持缩放和平移。

**布局：**

```
┌──────────────────────────────────────────────────────────────┐
│ 页面工作流 [5 个模块 · 2 个可交互] [缩放:100%] [刷新][导出]  │
│──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────┐  蓝线   ┌─────────────┐                    │
│  │ 🧩 导航栏   │─────────→│ 🖼️ Hero区  │                    │
│  │ 类型: header │         │ 类型: section│                    │
│  │ 模块: nav   │         │ 包含: 按钮   │                    │
│  │ 状态: 已分析 │         │ 状态: 已分析 │                    │
│  └─────────────┘         └──────┬──────┘                    │
│                                 │ 橙线                       │
│                                 ↓ 按钮→注册页                │
│  ┌─────────────┐  蓝线   ┌─────────────┐                    │
│  │ 📄 卡片区   │─────────→│ 📝 页脚    │                    │
│  │ 类型: grid  │         │ 类型: footer │                    │
│  │ 模块: cards │         │ 模块: links  │                    │
│  │ 包含: 3个卡片│         │ 含后端: API  │                    │
│  └─────────────┘         └─────────────┘                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**模块卡片设计（每个卡片 240px 宽 × 140px 高）：**

```
┌──────────────────────────┐
│ 🧩 导航栏                │ ← 标题行：图标 + 模块名称
│──────────────────────────│
│ 类型: header             │ ← 模块类型
│ 选择器: nav.header       │ ← CSS 选择器
│ 包含元素: 4个链接, 1个按钮 │ ← 子元素摘要
│──────────────────────────│
│ 交互: 无                 │ ← 交互类型（无/跳转/弹窗/提交）
│ 后端连接: 无             │ ← 后端关系
│──────────────────────────│
│ ● 已分析  📎 2个样式     │ ← 状态指示 + 样式数量
└──────────────────────────┘
```

**模块卡片信息字段：**

| 字段 | 来源 | 必要性 |
|---|---|---|
| 图标 | 自动（按类型分配） | 必须 |
| 模块名称 | 语义分析（nav→导航栏、section→内容区） | 必须 |
| 类型 | HTML 标签（header/nav/section/footer/button 等） | 必须 |
| 选择器 | 自动生成的 CSS 选择器 | 必须 |
| 包含元素 | 子元素数量和类型摘要 | 必须 |
| 交互类型 | 事件监听器 / href / onclick 分析 | 必须 |
| 后端连接 | fetch/axios/action 分析 | 有则显示 |
| 状态 | 已分析 / 待分析 / 用户标记 | 必须 |
| 样式数量 | 关联的 CSS 规则数量 | 可选 |

**连接线设计：**

| 连接类型 | 线条样式 | 颜色 | 含义 |
|---|---|---|---|
| 页面流向 | 实线 + 箭头 | 蓝色 (#4fc3f7) | 页面从上到下的阅读/浏览顺序 |
| 交互跳转 | 虚线 + 箭头 | 橙色 (#ffb74d) | 点击后跳转到另一个页面/模块 |
| 后端连接 | 点线 + 箭头 | 绿色 (#81c784) | 前端元素调用后端接口 |
| 依赖关系 | 细实线 | 灰色 (#4a4a5e) | 样式/数据依赖 |
**连接关系定义：**

以个人作品集网站为例，正确的模块连接关系如下：

**模块列表：**

| 模块 | 类型 | 说明 |
|---|---|---|
| 导航栏 | header | 顶部导航，含首页/作品/关于/联系四个链接 |
| Hero区 | section | 横幅区域，含标题和"查看作品"按钮 |
| 作品卡片区 | grid | 3张作品卡片，可点击查看详情 |
| 页脚 | footer | 底部版权和链接，含联系表单 |

**连接关系：**

| 起点 | 终点 | 连接类型 | 线条样式 | 触发条件 |
|---|---|---|---|---|
| 导航栏 | Hero区 | 页面流向 | 蓝色实线+箭头 | 页面从上到下的阅读顺序 |
| Hero区 | 作品卡片区 | 页面流向 | 蓝色实线+箭头 | 页面从上到下的阅读顺序 |
| 作品卡片区 | 页脚 | 页面流向 | 蓝色实线+箭头 | 页面从上到下的阅读顺序 |
| 导航栏"首页" | Hero区 | 交互跳转 | 橙色虚线+箭头 | 点击"首页"链接 |
| 导航栏"作品" | 作品卡片区 | 交互跳转 | 橙色虚线+箭头 | 点击"作品"链接 |
| 导航栏"联系" | 页脚 | 交互跳转 | 橙色虚线+箭头 | 点击"联系"链接 |
| Hero区"查看作品"按钮 | 作品卡片区 | 交互跳转 | 橙色虚线+箭头 | 点击按钮 |
| 作品卡片1 | 详情页(子页面) | 交互跳转 | 橙色虚线+箭头 | 点击卡片 |
| 作品卡片2 | 详情页(子页面) | 交互跳转 | 橙色虚线+箭头 | 点击卡片 |
| 页脚"发送"按钮 | API/submit | 后端连接 | 绿色点线+箭头 | 提交联系表单 |

**连接线路径规则：**

- 页面流向（蓝色）：从上到下，垂直方向，居中连接
- 交互跳转（橙色）：从起点右侧连接点出发，到目标左侧连接点，用贝塞尔曲线
- 后端连接（绿色）：从起点底部连接点出发，到目标顶部连接点
- 所有连接线不得交叉重叠
- 如果多条线连接到同一个目标，线条要在水平方向错开，避免重叠


**交互：**

| 操作 | 行为 |
|---|---|
| 拖拽模块卡片 | 自由移动位置，松手后保持新位置 |
| 鼠标滚轮 | 画布缩放（50%-200%） |
| 拖拽画布空白处 | 平移画布 |
| 单击模块 | 预览区滚动到该区域 + 高亮 |
| 右键模块 | 弹出菜单：编辑连接、AI 分析、删除标记、查看代码 |
| 双击模块 | 选中该元素 → 属性面板显示属性 |
| 拖拽模块边缘的连接点 | 创建/修改连接线 |
| 点击连接线 | 选中 → 可删除或修改 |
| 点击 [刷新] | 重新分析页面 → 更新拓扑图 |
| 点击 [缩放] | 重置到 100% |
| 点击 [导出] | 导出拓扑图为 SVG/PNG/JSON |

**画布控制栏：**

```
┌──────────────────────────────────────────────────────┐
│ 页面工作流 │ 5个模块 · 2个可交互 │ [-] 100% [+] │ 适应 │ 刷新 │ 导出 │
└──────────────────────────────────────────────────────┘
```

| 按钮 | 操作 |
|---|---|
| [-] | 缩小画布 |
| 100% | 显示当前缩放比例，点击重置 |
| [+] | 放大画布 |
| 适应 | 自动调整缩放使所有模块可见 |
| 刷新 | 重新分析页面 |
| 导出 | 导出为图片/JSON |

**画布缩放和拖拽的实现要求：**

- 整个画布是一个可变换的容器（CSS transform: scale() + translate()）
- 每个模块卡片是绝对定位的 div，可自由拖拽
- 连接线用 SVG 绘制，跟随模块位置实时更新
- 缩放时连接线粗细和模块大小按比例调整

**空状态：**

- 未打开项目时：显示"打开项目后自动生成页面工作流"
- 正在分析时：显示"正在分析页面结构..." + 加载动画
- 分析完成：显示完整拓扑图
- 页面为空（白板）：显示"添加元素后自动生成工作流"
